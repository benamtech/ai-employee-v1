## Problem

The repository already had a strong computation-first protocol, genuine software-invariant hypergraphs, candidate graphs, proof classes, and an important correction: a spectral object can be real proof of a precisely stated formal-model property.

The system was nevertheless operationally incomplete. Method authority still depended too heavily on prose and string-presence governance. No repository-owned compiler extracted facts, selected admissible representations, verified transformations, created a task capsule, enforced pre-edit chronology, verified a claim certificate, or bound predictions to outcomes.

Trace013 replaces that gap with executable infrastructure.

## Proof model

```text
P0 representation calculation or hypothesis
P1 verified property of an explicit formal model
P2 verified model-to-repository correspondence
P3 exact-candidate executable evidence
P4 external or production acceptance
```

The spectral verifier independently recomputes the normalized-operator residual and rejects mismatched model, certificate, correspondence, assumptions, tolerance, or excluded-claim boundaries. A valid eigenpair is real P1 proof of the declared operator property. It does not silently prove runtime correctness, engineering causality, or production acceptance.

## Representation plurality

The registry makes non-language representations first-class. Always-on dialects are facts, authority DAG, dependency graph, invariant hypergraph, correspondence, task diffusion, effect frontier, and experiment design.

The first implemented specialized dialect is spectral hypergraph analysis. The following are registered for later evidence-triggered implementation rather than falsely simulated now:

- transition systems and model checking;
- SMT constraints;
- e-graphs and equality saturation;
- queueing and control state;
- Koopman/DMD episodes;
- simplicial/cellular topology and Hodge operators;
- code-change embeddings;
- longitudinal trajectory episodes.

Natural language is generated as an interoperability and audit view. It is not the mandatory reasoning substrate.

## Experiment and calibration

The compiler records predictions before implementation and binds outcomes afterward. Default predictions cover:

- changed-file impact recall;
- hard-hyperedge proof-obligation coverage;
- authority-chain consistency;
- evidence non-promotion.

A six-case retrospective harness evaluates Trace007 through Trace012. It compares graph-plus-hypergraph task diffusion against lexical path retrieval for changed-source recall, required-test recall, total recall, context precision, and relevant invariant edges. Exact-head CI executes the harness. The report intentionally classifies causal engineering improvement as unestablished pending prospective matched tasks.

## Higher-order effects

First order:
- agents receive bounded repository facts and a task capsule instead of relying on prompt memory;
- formal certificates and software evidence become machine-distinct.

Second order:
- context can be queried on demand rather than frontloaded;
- changed paths, proof obligations, and evidence ceilings become enforceable transaction state;
- different agents share one vendor-neutral development protocol.

Third order:
- completed experiments create trustworthy trajectory episodes for later Koopman, queueing, or learned retrieval models;
- representations can evolve as dialects with explicit transformations rather than one universal ontology;
- small trusted verifiers reduce dependence on trusted coding agents.

Fourth order:
- engineering-method changes can be evaluated prospectively against baseline tasks;
- product-line, support, audit, and release policies can consume proof-carrying software transactions;
- the repository can progressively move from prompt-native to compiler-native agentic engineering without falsely claiming autonomous production acceptance.

## Nonclaims

Trace013 does not establish:

- causal superiority over the previous protocol;
- completeness of all possible repository facts or invariants;
- implementation of every registered future dialect;
- managed database, provider, target-host, accessibility, pilot, deployment, or production acceptance.

Those remain explicit experiments or higher evidence classes.
