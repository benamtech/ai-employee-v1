## Database evidence decision

Routine development does not require a live Supabase target. Production-shaped local/CI PostgreSQL proves:

- complete migration ledger;
- constraints, RLS, grants, functions, and negative isolation;
- backfills/existing-row compatibility;
- concurrency, races, compare-and-swap, rollback, and retry behavior;
- migration hashes and deterministic diagnostics.

Disposable managed Supabase remains required when platform-specific Auth, Realtime, Storage, Data API, advisor, security-sensitive browser behavior, suspected platform differences, new release migration classes, or the final release candidate is material. Production is never the routine test target.
