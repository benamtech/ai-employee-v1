# Generated Bespoke Software Strategy

Status: active Trace015 strategy artifact  
Evidence: P0/P2 planning only; no P4 acceptance claimed.

AMTECH should be understood as generated bespoke software for non-technical businesses. Each customer should feel like they received a custom operating system powered by AI employees or swarms, while the owner never touches API keys, schemas, providers, or infrastructure.

The immediate product/UI correction is to stop treating the owner surface as an approval-card dashboard. Approval remains a governed boundary, but the surface should materialize work-specific business apps: estimate desk, dispatch board, customer reply room, bookkeeping close, inventory cockpit, proof timeline, and owner intelligence brief.

Trace015 retains the 64D map and hypervector in machine-readable artifacts. The best first next steps are:

1. Generate multi-vertical Business OS blueprints.
2. Generate bespoke UI-variant drafts behind EmployeeExperienceModelV1.
3. Generate connector/effect/proof obligation drafts and block all P4 claims until live evidence exists.

This connects directly to WS-01 through WS-09: repository truth, connector authority, database authority, host custody, owner/channels, golden work, commercial ambiguity, recovery/release, and human surface/pilot.

## Current External Pattern Inputs

These are research inputs, not repository authority:

- OpenAI AgentKit points toward visual multi-agent workflows, connector registries, embedded agentic UI, and evals as one integrated agent-building stack: https://openai.com/index/introducing-agentkit/
- Anthropic computer-use guidance reinforces human checkpoints, scoped permissions, monitoring, logging, and treating external UI/web/email content as untrusted: https://claude.com/blog/best-practices-for-computer-and-browser-use-with-claude
- Google Cloud's agentic patterns place human-in-the-loop checkpoints directly inside agent workflows for critical approvals and subjective judgment: https://docs.cloud.google.com/architecture/choose-design-pattern-agentic-ai-system
- Cloudflare's agentic HITL docs show approvals can happen at MCP request, workflow, and connector-call layers: https://developers.cloudflare.com/agents/concepts/agentic-patterns/human-in-the-loop/
