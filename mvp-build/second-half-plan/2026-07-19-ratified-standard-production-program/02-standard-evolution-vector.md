# Standard Evolution Vector

Status: active explanatory companion  
Machine authority: `../../../validation/standard-v0.2-evolution-vector.json`

## Definition

The Standard evolution state is an n-dimensional vector where each dimension connects one original normative concern to:

- ratified v0.2 clauses;
- current implementation references;
- explicit supersession or research references;
- remaining blockers;
- semantic motion since the original baseline.

For v0.2, `n = 16`.

The direction basis is:

```text
[expansion, satisfaction, narrowing, destructive_modification, appendix, reorientation]
```

Each component is normalized to `[0,1]` and may coexist with another component. For example, connector architecture both expands the original Standard and reorients provider-specific OAuth assumptions into one managed setup protocol.

Velocity is the magnitude of normative and implementation movement since the original baseline. It is not:

- percent complete;
- confidence;
- code quality;
- production-readiness probability;
- permission to waive a hard gate.

## Step-back interpretation

The original Standard was not fundamentally wrong. Its strongest ideas—AI labor rather than chat, assignment scope, separate relationships, proof-bearing effects, and human authority—remain the center of v0.2.

The principal corrections are:

1. **Satisfaction:** current code now implements much of the original authority/effect ontology.
2. **Reorientation:** MCP Apps, current OAuth security, AG-UI's actual role, and standard supply-chain attestations changed the correct interoperability direction.
3. **Narrowing:** live database use is a release/platform gate rather than the routine development loop.
4. **Expansion:** connector manifests, engineering execution rules, protocol layering, and document control were underspecified.
5. **No destructive motion:** no hard protection was silently removed.

## Hard rules

- A vector cannot override a Boolean safety, authority, privacy, accounting, or deployment gate.
- Any non-zero destructive modification requires explicit human approval and loss analysis.
- Every dimension must remain grounded in original clauses and implementation or supersession references.
- Velocity changes only when normative or implementation state changes, not when prose is rewritten.
